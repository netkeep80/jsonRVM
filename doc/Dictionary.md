#### Контекст исполнения {#exec_ctx}
Это окружение для исполнения [**сущности отношения**](#ent_rel) в процессе работы РВМ. [**Модель отношений**](#rm) всегда выполняется внутри контекста исполнения.
Функцией контекста исполнения является предоставление доступа через [**местоимения**](#pronoun) к текущей исполняемой [**проекции сущности**](#ent_view).
По своей структуре контекст исполнения является [**проекцией сущности**](#ent_instance) в [**дереве контекстов исполнения**](#ctx_tree).
Текущий контекст исполнения обозначается через \$

#### Контекстный объект {#ctx_obj}
Местоимение для ссылки на объект контекста исполнения
Обозначается через \$obj

#### Контекстное отношение {#ctx_rel}
Значение экземпляра отношения контекста исполнения РВМ
т.е. на текущее значение контекстной сущности
Местоимение для ссылки на контекстное отношение обозначается через \$rel

#### Контекстная связь {#ctx_ref}
Это связь нижележащего контекста исполнения с вышестоящим. Эта связь хранится в структуре данных контекста исполнения и используется программистом для доступа к вышестоящим уровням контекста исполнения через [**местоимения**](#pronoun).

#### Контекстный субъект {#ctx_sub}
Местоимение для ссылки на субъект контекста исполнения РВМ
Обозначается через \$sub

#### Контекстная сущность {#ctx_ent}
[**Местоимение**](#pronoun) для ссылки на [**сущность**](#ent), проекцией которой является текущий [**контекст исполнения**](#exec_ctx) РВМ
Обозначается через \$ent

#### Корневой контекст исполнения {#root_ctx}
Контекст исполнения для которого вышестоящим является он сам

#### Корень модели отношений {#rm_root}

## Л

#### Лямбда вектор {#}
Массив json значений, которое необходимо исполнить RVM в текущем контексте, при этом последовательно используется текущее значение сущности

#### Лямбда cтруктура {#}
Множество пар "имя поля": значение поля, где:
* значение поля - иерархический путь к json значению, которое необходимо исполнить RVM в текущем контексте
* имя поля - иерархический путь к json значению которое необходимо вычислить

Вычисление проекций множества пар должно происходить параллельно, в многопоточном режиме

#### Объект {#obj}
Это определенная [**сущностью объекта**](#ent_obj) вещь, явление или процесс на которую направлена определённая [**сущностью отношения**](#ent_rel) деятельность [**субъекта**](#sub); при этом, в качестве объекта может выступать и сам [**субъект**](#sub).


#### Объективация {#objectification}
В зависимости от контекста употребления:
1. В [**МО**](#rm) - Использование сущности в качестве сущности объекта

Pronunciation

Noun
objectification (countable and uncountable, plural objectifications)

The process of objectifying something.
Synonyms
objectivization
reification
thingification

#### Сущность {#ent}
Сущность это определённый [**сущностью отношения**](#ent_rel) принцип взаимодействия [**субъекта**](#sub) с [**объектом**](#obj), описанный [**триплетом**](#triplet):

#### Сущность отношения {#ent_rel}
Это [**сущность**](#ent) определяющая принцип взаимодействия [**субъекта**](#sub) c [**объектом**](#obj)

Пример: сущность отношения 1+2 есть +

1. Отношение - это ассоциация или "связь" между двумя сущностями.
2. Взаимодействие 3х сущностей - субъекта, объекта и связи между ними
3. Описание взаимодействия между субъектом, объектом и связью.
4. Связь качество которой характеризует [**сущность**](#ent) выступающая в роли отношения
5. ОТНОШЕНИЕ — связь между некоторой сущностью и тем, что с ней соотнесено.

итого 1-3-9 :
1. сущность (это ОПРЕДЕЛЁННОЕ отношение субъекта к объекту)
3. субъект, отношение, объект
9. субъект (субъект как объект) - субъективно (субъек как субъект) - субъективация ( субъект как отношение)
   отношение - относительно - соотнесение
   объект - объективно - объективация
