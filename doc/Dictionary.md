## Дерево контекстов исполнения {#ctx_tree}
[**Контексты исполнения**](#exec_ctx) РВМ образуют многоуровневую иерархию исполнения экземпляров сущностей посредством связей двух типов: [**связью с предком**](#ancestor_link) и [**контекстной связью**](#ctx_ref).

## Контекст исполнения {#exec_ctx}
Это окружение для исполнения [**сущности отношения**](#ent_rel) в процессе работы РВМ. Модель отношений всегда выполняется внутри некоего контекста.
Функцией контекста исполнения является предоставление доступа через [**местоимения**](#pronoun) к текущему исполняемому экземпляру сущности.
По своей структуре контекст исполнения является [**экземпляром сущности**](#ent_instance) в [**дереве контекстов исполнения**](#ctx_tree).
Текущий контекст исполнения обозначается через \$

## Контекстный объект {#ctx_obj}
Местоимение для ссылки на объект контекста исполнения
Обозначается через \$obj

## Контекстное отношение {#ctx_rel}
Местоимение для ссылки на значение экземпляра отношения контекста исполнения РВМ
т.е. на текущее значение контекстной сущности
Обозначается через \$rel

## Контекстная связь {#ctx_ref}
Это связь нижележащего контекста исполнения с вышестоящим. Эта связь хранится в структуре данных контекста исполнения и используется программистом для доступа к вышестоящим уровням контекста исполнения через [**местоимения**](#pronoun).

## Контекстный субъект {#ctx_sub}
Местоимение для ссылки на субъект контекста исполнения РВМ
Обозначается через \$sub

## Контекстная сущность {#ctx_ent}
[**Местоимение**](#pronoun) для ссылки на [**сущность**](#ent), проекцией которой является текущий [**контекст исполнения**](#exec_ctx) РВМ
Обозначается через \$ent

## Корневой контекст исполнения {#root_ctx}
Контекст исполнения для которого вышестоящим является он сам

## Л

## Лямбда вектор {#}
Массив json значений, которое необходимо исполнить RVM в текущем контексте, при этом последовательно используется текущее значение сущности

## Лямбда cтруктура {#}
Множество пар "имя поля": значение поля, где:
* значение поля - иерархический путь к json значению, которое необходимо исполнить RVM в текущем контексте
* имя поля - иерархический путь к json значению которое необходимо вычислить

Вычисление проекций множества пар должно происходить параллельно, в многопоточном режиме

## Местоимение {#pronoun}

## Проецирование сущности {#view}

## Переменные контекста исполнения {#ctx_vars}

2. [**контекстный субъект**](#ctx_sub)
3. [**контекстное отношение**](#ctx_rel)
4. [**контекстный объект**](#ctx_obj)

## Связь с предком {#ancestor_link}

## Субъект {#sub}

## Сущность {#ent}
В зависимости от контекста использования:
1. В [**Модели отношений**](#rm) - [**Экземпляр отношения**](#rel_instance) между [**субъектом**](#sub) и [**объектом**](#obj)
2. В РВМ - [**Контекстная сущность**](#$ent)
